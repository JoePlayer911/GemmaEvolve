# OpenEvolve Updates - 2026-03-11

## Iteration Checkpoint & Resume Capabilities
We introduced robust, state-saving enhancements to the OpenEvolve framework today to protect your long-running benchmark sessions from critical interruptions. Instead of throwing out all evolutionary progress when closing a run mid-way, you can now seamlessly resume exactly where you left off. 

### Key Capabilities Added
1. **Configurable Save Frequency.** Through the newly added `--checkpoint-interval` CLI flag (and `--save-freq` flag on the benchmarking script), you tell OpenEvolve exactly how often to serialize the progress snapshot down to disk (island configurations, score metrics, program definitions). 

2. **Auto-Resume Integration.** `run_completion_benchmark.py` will now automatically detect `checkpoints/checkpoint_*` directories inside previously attempted problem runs. If you restart an interrupted test, it parses the latest saved iteration and calculates how much further it needs to compute from that point to reach your global limit!

3. **Infinite Extension.** Want to throw 800 extra iterations on problems that were getting close to solving? You can simply launch the benchmark tool using a higher `--iterations` flag. If it was resting continuously on an existing `checkpoint_800`, the tool effortlessly calculates the mathematics and asks the OpenEvolve controller to run `max_iterations - latest_checkpoint` *more* evolutionary loops. 

> *Note: By default, the `max_iterations` limit remains 800 and the checkpoint frequency is every 100 loops.*
